Movie Database

This movie database I constructed used three seperate CSV's. One for the movie data, one for the rating data, and one for the user data. Using these, I created a list of movies and users, then linked the two using the user's ratings of the movie. Every instance of a user and movie is a link back to that user or movie's page. Also, I was able to create a simple property to show the top 20 movies with the highest ratings with at least 10 ratings.


Using Django to create this web app, this project is what first ignited the passion for web development. This was also the first web app I created with a nav bar. Once I was able to click on everything and be able to navigate this app, I was hooked. While this is a very basic web app, this is one of my favorites because it opened the door to web development for me. I am very proud of this one.

  • Date: October 2015

Q&A

This project was a team assignment at The Iron Yard. A partner and I were given 4 days to construct a web app in the style of Stack Overflow, complete with tags. Users can ask and answer questions, view all questions that have been asked, and view their profile to be able to view all questions the user has asked and answered. We also included a "Score" feature for each answer, so that the answer with the highest score is displayed first. Each user is also assigned a score based upon the ratings of the answers they have provided. All users, except for the answerer, are allowed to vote once on each answer.


This assignment sealed my fate, so to speak. Once I finished this assignment, I knew I wanted to pursue a career in web development. All of the intricacies involved with the rating system for this project were just lovely, and I enjoyed figuring them out. This assignment was also where I was introduded to Materialize. This CSS framework became my primary tool for design after being exposed to it.

  • Date: November 2015

Cater'd

This was my final project at The Iron Yard. I worked with another student on this for 4 weeks and produced a result I am supremely proud of. Cater'd offers users a choice of being a Chef or a Client. Chefs can create events for Clients to attend, and Clients can create events for Chefs to provide for. As a former cook, this project is near and dear to my heart, and I was very very pleased with the result.


This assignment used Django for the construction, and we were able to deploy live to the web with Amazon Web Services. We also were able to incorporate Elasticsearch to allow for a search function. Materialize was used for the front-end design as well. I genuinely enjoyed creating this app, and look forward to creating many more in my career.

  • Date: November/December 2015
